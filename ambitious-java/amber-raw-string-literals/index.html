随着JDK 12(2019年3月)的推出, Java终于迎来了”生”字符串字面量(Raw String Literals).

为什么要新引入”生”字符串字面量?

对于多行字符串拼接问题, Java之前做的并不出色, 如果字符串过长, 很容易造成”拼接地狱”问题; 类似的还有Java正则, 由于正则模式使用字符串表示, 如果正则过于复杂或转义字符很多, 将严重影响代码可读性以及可维护性. Java中”生”字符串字面量的出现, 有助于解决这些问题, 提高代码可读性以及可维护性.
